使用 ARP 配置 IP 地址

相关型号:DocuPrint P268 d / DocuPrint P268 dw

即使您不能使用 BRAdmin 应用程序，并且您的网络中没有 DHCP 服务器，您也可以使用地址解析协议 (ARP) 命令。ARP 命令适用于安装有 TCP/IP 协议的 Windows® 系统和 UNIX® 系统。若要使用 ARP，请输入以下命令：

arp -s ipaddress ethernetaddress

ipaddress

其中，ethernetaddress 是打印服务器的 MAC 地址 (以太网地址)，ipaddress 是打印服务器的 IP 地址。

例如：

Windows® 系统

Windows® 系统要求在 MAC 地址 (以太网地址) 的每个数字之间使用破折号 "-"。

arp -s 192.168.1.2 00-80-77-31-01-07

ping 192.168.1.2

UNIX®/Linux 系统

通常情况下，UNIX® 和 Linux 系统要求在 MAC 地址 (以太网地址) 的数字之间使用冒号 “:”。

arp -s 192.168.1.2 00:80:77:31:01:07

ping 192.168.1.2

您必须与打印服务器处于相同的网段 (即：在打印服务器和操作系统之间不能有路由器) 才能使用 ARP 命令。

如有路由器，可使用 BOOTP 或其他方法输入 IP 地址。如果您的管理员已配置系统以使用 BOOTP、DHCP 或 RARP 传送 IP 地址，您的 Fuji Xerox 打印服务器可从任何一个 IP 地址分配系统中接收 IP 地址，这样您就无需使用 ARP 命令。ARP 命令仅可使用一次。为了安全起见，当您使用 APR 命令成功配置 Fuji Xerox 打印服务器的 IP 地址时，不得再次使用此 APR 命令更改地址。打印服务器将忽略此类尝试。若要再次更改 IP 地址，请使用网络基本管理，TELNET (使用 SET IP ADDRESS 命令)，或在打印服务器上执行出厂设置，然后将允许您再次使用 APR 命令。
