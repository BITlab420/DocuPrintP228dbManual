使用 BOOTP 配置 IP 地址

相关型号:DocuPrint P268 d / DocuPrint P268 dw

在使用引导协议 (BOOTP) 配置 IP 地址前，必须将设备的引导方式设置为 BOOTP。若要更改引导方式，请使用设备操作面板上的Network (网络)菜单或 BRAdmin 应用程序之一 (BRAdmin Light)。

BOOTP 是逆向地址解析协议 (RARP) 的替代，具有可配置子网掩码和网关的优势。若要使用 BOOTP 设置 IP 地址，确保 BOOTP 已安装且在主机上运行 (应作为真实服务出现在主机的 /etc/services 文件中，输入 man bootpd 或参阅您的系统文件了解详情)。BOOTP 通常经由 /etc/inetd.conf 文件启动，因此您可能需要删除此文件中 bootp 输入前的 "#" 将其启用。例如，/etc/inetd.conf 文件中的典型 bootp 输入为：

#bootp dgram udp wait /usr/etc/bootpd bootpd -i

根据系统的不同，此输入可能被称为 "bootps" 而非 "bootp"。

若要启用 BOOTP，使用编辑器删除 "#" (如果没有 "#"，说明 BOOTP 已启用)。然后，请编辑 BOOTP 配置文件 (一般为/etc/bootptab) 并输入打印服务器的名称、网络类型 (1 为以太网)、MAC 地址 (以太网地址) 以及 IP 地址、子网掩码和网关。但是这种操作的格式仍无统一标准，所以您必须参见您的系统文件，以决定如何输入此类信息 (在 bootptab 文件中，许多 UNIX® 系统也有模板例子，您可以将其作为参考)。一些典型的/etc/bootptab例子包括：(对于无线网络，下例中的 "BRN" 为 "BRW"。)

BRN310107 1 00:80:77:31:01:07 192.168.1.2

和：

BRN310107:ht=ethernet:ha=008077310107:\ip=192.168.1.2:

如未在配置文件中包括下载文件名，某些 BOOTP 主机软件实现不会响应 BOOTP 请求。如果出现此情况，在主机上创建一个空文件，在配置文件中指定此文件的名称及其路径。

使用 RARP 时，打印服务器将在打开设备电源时从 BOOTP 服务器加载其 IP 地址。
